# Data Directory Guide

This folder stores local project data.

Large real datasets should not be committed to Git.

## Folders

### raw

Original downloaded files.

Rules:

- Do not manually edit raw files
- Keep filenames traceable
- Record source details in a manifest

### interim

Cleaned or partially transformed files.

Examples:

- clipped boundary files
- normalized coordinate files
- temporary geospatial joins

### processed

Machine-learning-ready files.

Examples:

- feature tables
- train/test splits
- grid-level flood-risk dataset

### external

Reference files from third parties that are not generated by this project.

Examples:

- administrative boundaries
- reference lookup tables

### samples

Small generated demo datasets safe to commit.

Used for tests, examples, and the Streamlit MVP.
